#+title: Homelab Infrastructure

* Overview
Homelab infrastructure on Proxmox, managed with OpenTofu, Ansible, and NixOS.

** Architecture
- *Proxmox VE* for compute (LXC containers and VMs)
- *OPNsense* firewall for network segmentation
- *HashiCorp Vault* for secrets and PKI certificates
- *Authentik* for identity management
- *NixOS LXC containers* deployed via =deploy-rs= (replacing Ansible for new machines)

** Services
*** Ansible-managed (Debian LXC)
- Vault (ID 200)
- Root CA (ID 201)
- Authentik (ID 202)

*** NixOS-managed
- Gitea (ID 203) — Git hosting
- Homepage (ID 204) — Dashboard with service status and widgets

** Nix Flake Structure
The flake uses =flake-parts= for clean separation of per-system and non-per-system outputs.

| Output                  | Description                              |
|-------------------------+------------------------------------------|
| =devShells.default=     | Dev environment (tofu, ansible, sops...) |
| =packages.lxc-template= | NixOS LXC tarball for Proxmox            |
| =nixosConfigurations=   | NixOS machine configs                    |
| =deploy.nodes=          | deploy-rs deployment targets             |

* NixOS LXC Setup

** Building the Template
#+begin_src bash
nix build .#lxc-template
#+end_src

The tarball is uploaded to Proxmox via =proxmox_virtual_environment_file= in Terraform.

** Creating a Container
Terraform creates NixOS LXC containers with =type = "nixos"=, which tells Proxmox to inject network config (IP, gateway) via systemd-networkd.

#+begin_src bash
cd terraform && tofu apply
#+end_src

** Extracting SSH Keys
After =tofu apply=, extract the NixOS machine keys:

*** bash/zsh
#+begin_src bash
cd terraform
tofu output -raw nixarr_private_key > ../.secrets/nixarr_key && chmod 600 ../.secrets/nixarr_key
#+end_src

*** nushell
#+begin_src nu
cd terraform
tofu output -raw nixarr_private_key | save -f ../.secrets/nixarr_key; chmod 600 ../.secrets/nixarr_key
#+end_src

** Updating sops Keys (after container recreation)
If a container is recreated, its SSH host key changes. Update the sops age key:

#+begin_src bash
./scripts/sops-helper.sh update-keys nixarr 192.168.1.54
#+end_src

** Deploying Configuration
#+begin_src bash
nix run github:serokell/deploy-rs -- -s .
#+end_src

deploy-rs pushes NixOS configuration over SSH with automatic rollback on failure.

* Secrets Management

Secrets are managed with =sops-nix= using age encryption. Each machine decrypts secrets at activation time using its SSH host key.

** Managing Secrets
#+begin_src bash
./scripts/sops-helper.sh set nixarr nixarr_role_id "value"
./scripts/sops-helper.sh update-keys nixarr 192.168.1.54
#+end_src

** Vault Agent
NixOS machines run =vault-agent= for automatic TLS certificate issuance via Vault's PKI engine. AppRole credentials are delivered through sops-nix. The Vault CA chain is committed to =certs/= and added to the system trust store.

* Firewall
** Manual Installation Procedure
The OS must be manually installed to the virtual hard disk. This is a one-time operation.

1. After applying the Terraform configuration, start the VM and open the Proxmox console.
2. The VM will boot into a live environment from the ISO. It will prompt for interface assignment and then show a =Login:= prompt.
3. To start the installation, log in with the following credentials:
   - *Username:* =installer=
   - *Password:* =opnsense=
4. Follow the on-screen installation wizard. Select =Install (UFS)=, choose the virtual disk (=da0=), and confirm that you want to erase the disk.
5. Once the installation is complete, reboot the system from the wizard.
6. Update your IaC to reflect the new state. Modify the =proxmox_virtual_environment_vm= resource:
   - Change the boot order to prioritize the hard disk: =boot_order = ["scsi0"]=
   - Comment out or remove the entire =cdrom= block. The ISO is no longer needed.
7. Run =terraform apply= to persist these changes. The VM is now permanently installed.

** OPNsense Network and DNS Configuration
*** Initial Console Configuration (Post-Installation)
After the first boot from the hard disk, you must perform the initial setup via the console.

1.  *Assign Interfaces:* Choose option =1) Assign interfaces=. Match the MAC addresses shown to the hardware in the Proxmox UI to correctly identify which virtual NIC (=vtnet0= or =vtnet1=) corresponds to your WAN (=vmbr0=) and LAN (=vmbr1=).
2.  *Set LAN IP Address:* Choose option =2) Set interface IP address=. Select the LAN interface and configure it with a static IP address for your new protected network (e.g., =192.168.100.1/24=). Do not set an upstream gateway for the LAN.

*** Dnsmasq Configuration for DNS and DHCP
For simple, integrated DNS and DHCP, Dnsmasq is the correct tool.

1.  *Disable Conflicting Services:* To ensure a clean state, navigate to =Services -> Unbound DNS= and =Services -> ISC DHCPv4= and ensure both are disabled.
2.  *Enable Dnsmasq:* Navigate to =Services -> Dnsmasq DNS & DHCP -> General=.
    - Check the =Enable= box.
    - Set =Interface= to =LAN=.
3.  *Set the DNS Listen Port:*
    - In the =DNS= section, set =Listen port= to =53=.
      #+BEGIN_NOTE
      A value of =0= is supposed to default to 53, but was identified as the root cause of a critical failure where the DNS service would not start. Setting it explicitly to =53= is mandatory.
      #+END_NOTE
4.  *Configure DHCP Range:*
    - Go to the =DHCP ranges= tab.
    - Add a new range for the =LAN= interface (e.g., from =192.168.100.100= to =192.168.100.200=).
5.  *Set the Domain:*
    - Go back to the =General= tab.
    - Scroll down and set the =DHCP default domain= to your desired internal domain (e.g., =lab.lan=).
6.  *Apply Changes:* Click the =Apply= button. Your protected LAN now has fully functional DHCP and automatically registered DNS for its clients.

* Homepage Dashboard Setup

Homepage provides a dashboard with service status widgets for Proxmox, Gitea, Vault, and Authentik.

** Infrastructure (Terraform)
#+begin_src bash
cd terraform && tofu apply
#+end_src

This creates:
- LXC container (ID 204, IP 192.168.100.12)
- Proxmox API user =homepage@pve= with =PVEAuditor= role
- API token =homepage@pve!dashboard=

** Vault AppRole (machines/vault)
#+begin_src bash
cd machines/vault && tofu apply
#+end_src

Creates the =homepage= approle for certificate issuance.

** Extract SSH Key and Update Sops
#+begin_src bash
cd terraform
tofu output -raw homepage_private_key > ../.secrets/homepage_key && chmod 600 ../.secrets/homepage_key

# Update sops age key from container's SSH host key
./scripts/sops-helper.sh update-keys homepage 192.168.100.12
#+end_src

** Configure Secrets
#+begin_src bash
# Proxmox token (from terraform)
./scripts/sops-helper.sh set homepage proxmox_token_id "$(cd terraform && tofu output -raw homepage_proxmox_token_id)"
./scripts/sops-helper.sh set homepage proxmox_token_secret "$(cd terraform && tofu output -raw homepage_proxmox_token_secret)"

# Vault approle (from machines/vault)
./scripts/sops-helper.sh set homepage homepage_role_id "$(cd machines/vault && tofu output -raw homepage_role_id)"
./scripts/sops-helper.sh set homepage homepage_secret_id "$(cd machines/vault && tofu output -raw homepage_secret_id)"
#+end_src

** Deploy Gitea First (for API token)
Gitea automatically generates an API token for Homepage on first deploy:
#+begin_src bash
deploy .#gitea --skip-checks

# Get the generated token
TOKEN=$(ssh root@192.168.100.53 cat /var/lib/gitea/homepage-token | grep -oP 'created: \K.*')
./scripts/sops-helper.sh set homepage gitea_token "$TOKEN"
#+end_src

** Deploy Homepage
#+begin_src bash
deploy .#homepage --skip-checks
#+end_src

Homepage will be available at =http://192.168.100.12:8082=

* Known Issues

** Proxmox NixOS DNS Injection
Proxmox's =ostype: nixos= injects IP and gateway into systemd-networkd files but does not inject DNS or search domain. NixOS machines must set =networking.nameservers= and =networking.search= explicitly. This is handled by =machines/nixos/modules/network.nix=.

** terraform-validate in Nix Sandbox
The =terraform-validate= pre-commit hook fails inside the Nix sandbox (no network access). It is disabled in =nix flake check= but still runs as a git hook in the devShell.
